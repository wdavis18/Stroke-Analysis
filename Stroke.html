Introduction¶

Strokes happen when something "blocks blood supply to part of the brain or when a blood vessel in the brain bursts."(About Stroke | cdc.gov, 2022). In America, strokes are the fifth leading cause of death and the leading cause of long-term disability.(Neurosciences - Stroke Facts, n.d.). For reference, for every 40 seconds in America someone has a stroke and for every 3.5 minutes, someone dies of a stroke.(Neurosciences - Stroke Facts, n.d.). This research will provide insight on the predictors of a patient having a stroke.¶

The Stroke Prediction Dataset was obtained from Kaggle and contains the following variables.¶

(Stroke Prediction Dataset, 2021)¶

* id: unique identifier¶

* gender: "Male", "Female" or "Other"¶

* age: age of the patient¶

* hypertension: 0 if the patient doesn't have hypertension, 1 if the patient has hypertension¶

* heart_disease: 0 if the patient doesn't have any heart diseases, 1 if the patient has a heart disease¶

* ever_married: "No" or "Yes"¶

* work_type: "children", "Govt_jov", "Never_worked", "Private" or "Self-employed"¶

* Residence_type: "Rural" or "Urban"¶

* avg_glucose_level: average glucose level in blood¶

* bmi: body mass index¶

smoking_status: "formerly smoked", "never smoked", "smokes" or "Unknown"¶

* stroke: 1 if the patient had a stroke or 0 if not¶

Logistic Regression with Cross Validation¶

Logistic regression is a statistical model used for classification and predictive analytics. It "estimates the probability of an event occurring, such as voted or didn’t vote, based on a given dataset of independent variables. Since the outcome is a probability, the dependent variable is bounded between 0 and 1." (What Is Logistic Regression? | IBM, n.d.)¶

Since the model predicts the output based on 0 and 1 (or yes and no for our case), we use our label encoder to transform our response variable 'stroke' from "Yes" and "No', to "1" and "0" respectively.¶

"Cross validation (CV) is one of the technique used to test the effectiveness of a machine learning models, it is also a re-sampling procedure used to evaluate a model if we have a limited data." (.M, 2020)¶

A stratified sample will "divide a population into homogeneous subpopulations called strata (the plural of stratum) based on specific characteristics (e.g., race, gender identity, location, etc.). Every member of the population studied should be in exactly one stratum. Each stratum is then sampled using another probability sampling method, such as cluster or simple random sampling, allowing researchers to estimate statistical measures for each sub-population. Researchers rely on stratified sampling when a population’s characteristics are diverse and they want to ensure that every characteristic is properly represented in the sample." (Thomas, 2022)¶

To verify the accuracy of the model, a stratified k-fold is used to split the randomized data into training and testing datasets.¶

Extra Tree Classifier with Cross Validation¶

An Extra Tree Classifier is defined as "a type of ensemble learning technique which aggregates the results of multiple de-correlated decision trees collected in a “forest” to output it’s classification result." (GeeksforGeeks, 2020)¶

"Each Decision Tree in the Extra Trees Forest is constructed from the original training sample. Then, at each test node, Each tree is provided with a random sample of k features from the feature-set from which each decision tree must select the best feature to split the data based on some mathematical criteria (typically the Gini Index). This random sample of features leads to the creation of multiple de-correlated decision trees."(GeeksforGeeks, 2020)¶

To the increase the randomness of the sample, an extra tree classifier was implemented as a classification model to produce more accurate results. The model was trained and tested using 500, 1000, 1500, and 2000 tress, while also using stratified k-fold cross validation.¶

In [108]:
# Creating our training and sample data
np.random.seed(123)
skf = StratifiedKFold(n_splits=5,shuffle = True)
tree_count = [500,1000,2000,3000,5000,7000]
err = []
out = list()
for tree in tree_count:
  model_two = ExtraTreesClassifier(n_estimators=tree)
  for tr_index, te_index in skf.split(data_randomized, data_randomized['stroke']):
    x_train = data_randomized.iloc[tr_index]
    x_train =  x_train.iloc[:,:-1]
    x_train = enc.fit_transform(x_train)
    x_train = pd.DataFrame(x_train, columns=colnames)

    x_test = data_randomized.iloc[te_index]
    x_test =  x_test.iloc[:,:-1]
    x_test = enc.fit_transform(x_test)
    x_test = pd.DataFrame(x_test, columns=colnames)

    y_train = data_randomized['stroke'].iloc[tr_index] 
    y_test = data_randomized['stroke'].iloc[te_index]

    y_train = le.fit_transform(y_train)
    y_test = le.fit_transform(y_test)
    model_two.fit(x_train,y_train)
    yhat_test_two = model_two.predict(x_test)

    
    err.append(metrics.accuracy_score(y_test,yhat_test_two))
  out.append(f"Accuracy: {np.round((sum(err)/len(err)) * 100, 2)}% with {tree} trees")

Conclusion¶

In conclusion, both classification models produced similar results on the smaller dataset. Given that logistic regression is divdies the predictors into two categories, while decision tress divides them into multiple boundaries, it is expected that logistic regression will work just as well on the smaller dataset. If however, we were to have a larger number of explanatory variables within the dataset, the extra tree classification model will out perform and produce more accurate results.¶
